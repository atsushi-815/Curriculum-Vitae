## 🛠 Personal Projects / ライフワーク

### ポケモンカード対戦管理システム（AWS Lambda × Slack）
娘とのジムバトル参戦をより楽しむため、戦績の記録・分析を行うサーバーレスシステムを自作・運用しています。

- **技術スタック:** AWS Lambda, Python, Slack API (Bolt), Google Sheets API
- **実装内容:**
  - **対戦記録の自動化:** Slackから「名前・勝敗・デッキ」を入力し、Googleスプレッドシートへ自動蓄積。全角・半角の入力揺らぎを許容するフロントエンド（Slackインタフェース）を構築。
  - **データ分析機能:** `/pokeca-stats` コマンドにより、累計勝率やデッキ別の有利・不利をLambdaで即座に計算しSlackへフィードバック。
  - **定期バッチ処理:** EventBridgeを用いて毎週月曜朝に「先週のMVPデッキ」や通算勝敗を自動集計してSlack投稿。
- **ポイント:** 日常の課題をIaC（Serverless）で解決。単なる記録に留まらず、週次レポートによる振り返り（PDCA）の仕組みまでを構築・運用しています。
